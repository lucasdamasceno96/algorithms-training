# Algorithms Training: Path to Big Tech 🚀

This repository contains my daily practice of data structures and algorithms, implemented in both **Python** and **Go**. My goal is to master the fundamentals required for Cloud, DevOps, and Platform Engineering roles at top-tier technology companies.

## 🎯 Goal

- Practice 1 algorithm per day.
- Implement every solution in Python (for scripting/automation) and Go (for systems/performance).
- Build a progressive learning path where each topic reinforces the previous one.

## 🛠 Tech Stack

- **Python**: Focused on readability and rapid prototyping.
- **Go**: Focused on concurrency, types, and memory efficiency.

## 📂 Repository Structure

- `/python`: Python implementations and unit tests.
- `/go`: Go implementations using standard testing packages.

## 📈 Roadmap & Progress

Check the list below to track my evolution from basic arrays to complex graph traversals.

Preparar-se para Big Techs focando em Cloud e Platform Engineering exige um equilíbrio entre lógica pura e eficiência de recursos. Python é a linguagem "cola" da automação, enquanto Go é a linguagem de sistemas e infraestrutura moderna (Kubernetes, Terraform e Docker são escritos em Go).

## 📂 Estrutura do Repositório

Para um perfil de **DevOps/Platform**, a organização por **linguagem** é mais limpa. Isso permite que você configure ambientes isolados (como um `go.mod` na raiz da pasta Go e um `requirements.txt` ou `venv` na pasta Python) sem que um interfira no outro.

**Estrutura sugerida:**

```text
/algorithms-training
├── /python
│   ├── 01_reverse_string.py
│   └── 02_two_sum.py
├── /go
│   ├── 01_reverse_string.go
│   └── 02_two_sum.go
├── README.md
└── progress.md (opcional)

```
